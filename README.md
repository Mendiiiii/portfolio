  MAJOR.MINOR.PATCH
                                                                                
  - PATCH (1.0.3 → 1.0.4) — fixes, ajustes pequeños, copy, bugs                 
  - MINOR (1.0.x → 1.1.0) — nueva funcionalidad que no rompe lo existente. Por
  ejemplo: una nueva página de proyecto, una nueva sección, un feature nuevo    
  - MAJOR (1.x.x → 2.0.0) — cambio grande que rompe o reemplaza algo
  fundamental. Por ejemplo: rediseño completo, cambio de stack, restructura     
  total del portfolio
